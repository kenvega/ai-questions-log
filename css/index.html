Terms before properties

Understand the parts of a CSS Grid

Grid terminology describes the container, its children, and the invisible structure that places them.

HTML for this lesson

The direct children are grid items. The paragraph nested inside item B is not.

<div class="container">
  <div class="item">A</div>
  <div class="item">
    B <p>nested content</p>
  </div>
  <div class="item">C</div>
</div>

Container and items

The container owns the grid

Grid container: the element with display: grid. Grid items: only its direct children.

.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}

Green outline = container · colored boxes = items

Grid lines

Lines are numbered boundaries

A three-column grid has four vertical lines. A two-row grid has three horizontal lines. Placement properties refer to these numbers.

.item {
  grid-column: 2 / 4;
  grid-row: 1 / 2;
}

Column lines 1–4 · row lines 1–3

1234 123 starts at column line 2
ends at column line 4

Grid tracks

A track is the space between two lines

A complete column is a column track. A complete row is a row track. Track sizing properties control their width or height.

.container {
  grid-template-columns: 80px 1fr 1fr;
  grid-template-rows: 52px 52px;
}

Orange = one column track · purple = one row track

80px 52px 52px
column trackrow track

Grid cells

A cell is one grid unit

One cell is enclosed by two adjacent column lines and two adjacent row lines. It is the Grid equivalent of one table cell.

/* highlighted cell */
column lines: 2 → 3
row lines:    1 → 2

This cell sits between column lines 2–3 and row lines 1–2

one cell

Grid areas

An area is a rectangle of cells

An area may be one cell or many cells, but it must be rectangular and enclosed by four grid lines.

.item {
  grid-area: 1 / 1 / 3 / 3;
  /* row-start / column-start /
     row-end / column-end */
}

This area spans four cells from lines 1 to 3 on both axes

2 × 2 grid area

Explicit and implicit grids

Defined tracks versus generated tracks

Explicit tracks come from grid-template-*. If more items need space, Grid creates implicit tracks.

.container {
  grid-template-rows: 52px; /* explicit */
  grid-auto-rows: 36px;     /* implicit */
}

Solid row = explicit · dashed rows = implicit

Quick recap

Grid container
The element with display: grid.
Grid item
A direct child of the grid container.
Grid line
A numbered dividing line on the row or column axis.
Grid track
The space between two adjacent parallel lines.
Grid cell
One row track crossed with one column track.
Grid area
Any rectangular group of cells bounded by four lines.

Sketch layouts with named areas

Named areas rearrange these semantic elements without changing their source order.

  <main>main</main>
  <footer>footer</footer>
</div>
Starting point

Automatic placement

  display: grid;
  grid-template-columns: 100px 1fr;
}
/* no named areas yet */

Name the page regions
.layout {
  grid-template-columns: 90px 1fr;
  grid-template-rows: 35px 74px 30px;
  grid-template-areas:
    "header header"
    "nav    main"
    "footer footer";
}
.header { grid-area: header; }

Names place the regions; the columns/rows above still set their sizes — nav stays 90px while main fills the rest

headernavmainfooter
Use the template shorthand
.layout {
  grid-template:
    "header" 44px
    "main"   1fr
    "nav"    44px
    "footer" 38px
    / 1fr;
}
/* grid-template sets areas, rows + columns */

Header and nav are 44px; the footer is 38px

Keep in mind: this last layout shows main above nav, but the HTML order above never changed — nav still comes before main in the markup. Keyboard tabbing and screen readers follow that source order, not the visual one, so a sighted user and a keyboard user reach these two regions in a different sequence.
